# weekly-analysis

전주 대비 매출 변화를 분해(decompose)하고, AI가 원인을 가설화한 뒤 조치를 권장하는 워크플로우.

## When to use

- "지난 주 대비 매출이 왜 줄었어?" 같은 원인 분석 요청 시
- 데일리 리포트 후 "더 자세히 알아봐줘" 요청 시
- 정기 주간 브리핑 (금요일 마감, 월요일 리뷰)

## Prerequisites

- 최소 1개 채널 주문 데이터 수집 완료
- 원가 등록 권장 (없으면 매출이익률 분석 skip): `sppt cost list`

## Data Collection

아래 커맨드로 이번 주 + 지난 주 데이터를 수집합니다.

```bash
# 이번 주 채널별 요약
sppt report --by channel --period 7d --json

# 지난 주 채널별 요약
sppt report --by channel --period 7d --offset 7d --json
# TODO: --offset 플래그 구현 필요 (현재 미구현)
# 임시 대안: --from YYYY-MM-DD --to YYYY-MM-DD 사용
```

```bash
# 이번 주 상품별 요약 (top 10)
sppt report --by product --period 7d --top 10 --json

# 지난 주 상품별 요약
sppt report --by product --period 7d --offset 7d --top 10 --json
```

```bash
# 원가 정보 (공헌이익 계산용)
sppt cost list --json
```

```bash
# 정산 요약 — 채널별 수수료 실효율
sppt settlement summary --days 7 --json
sppt settlement summary --days 14 --json
```

> **참고**: `--offset` 플래그가 없는 경우 날짜를 직접 지정:
> ```bash
> sppt report --by channel --from 2026-04-14 --to 2026-04-21 --json
> sppt report --by channel --from 2026-04-21 --to 2026-04-28 --json
> ```

## Analysis Steps

### Step 1: 채널 믹스 분해 (수수료율 변화 원인)

수집한 채널별 데이터로 아래를 계산합니다:

```
실효수수료율(이번주) = Σ(채널_매출비중 × 채널_수수료율)
실효수수료율(지난주) = Σ(채널_매출비중 × 채널_수수료율)
변화량 = 이번주 - 지난주

원인 분해:
  채널믹스 효과 = Σ((이번주_비중 - 지난주_비중) × 지난주_수수료율)
  수수료율 변화 효과 = Σ(이번주_비중 × (이번주_수수료율 - 지난주_수수료율))
```

출력 예시:
```
📊 수수료율 분석 (이번 주 vs 지난 주)
──────────────────────────────────────────
지난 주 실효수수료율:  8.2%
이번 주 실효수수료율:  9.7%
변화:                +1.5%p

원인:
  채널 믹스 이동:     +1.3%p  (쿠팡 비중 +12%p)
  채널 내 수수료율:   +0.2%p  (변동 없음 or 카테고리 변화)

→ 수수료율 상승의 87%는 쿠팡 집중으로 인한 믹스 효과
```

### Step 2: 채널별 매출 분포 변화 분해

```
채널 점유율 변화 = 이번주_채널매출/이번주_총매출 - 지난주_채널매출/지난주_총매출

채널 매출 변화 = 볼륨 변화 + 단가 변화
  볼륨 효과   = (이번주_수량 - 지난주_수량) × 지난주_평균단가
  단가 효과   = (이번주_평균단가 - 지난주_평균단가) × 이번주_수량
```

출력 예시:
```
📊 채널별 매출 변화
─────────────────────────────────────────────
채널          지난주       이번주    변화    볼륨   단가
cafe24       850,000      620,000   -27%   -25%   -3%
coupang    2,150,000    2,890,000   +34%   +30%   +3%
smartstore   620,000      480,000   -23%   -20%   -4%
─────────────────────────────────────────────
합계       3,620,000    3,990,000   +10%

채널 점유율 변화:
  쿠팡: 59% → 72% (+13%p)  ← 수수료율 상승 주원인
  카페24: 23% → 16% (-7%p)
```

### Step 3: 상품별 기여 분해 (매출 변화 원인 귀속)

채널 변화 중 어떤 상품이 기여했는지:

```
채널 매출 변화 = Σ(상품별 매출 변화)
상품별 기여율 = 상품_매출변화 / 채널_총매출변화 × 100
```

출력 예시:
```
📦 쿠팡 매출 +740,000원 변화 원인
  [+] A상품: +520,000원 (70%)  — 수량 +35건
  [+] C상품: +380,000원 (51%)  — 신규 등록
  [-] B상품: -160,000원 (-22%) — 수량 -12건 (재고 부족 추정)
```

### Step 4: 매출이익률 분석 (원가 데이터 있을 때만)

원가가 등록된 SKU에 한해:

```
공헌이익률(이번주) = (매출 - 총변동비) / 매출
공헌이익률(지난주) = ...
변화 원인:
  상품 믹스 효과 = Σ((이번주_비중 - 지난주_비중) × 지난주_공헌이익률)
  이익률 변화 효과 = Σ(이번주_비중 × (이번주_공헌이익률 - 지난주_공헌이익률))
```

출력 예시:
```
💰 매출이익률 분석
──────────────────────────────────
지난 주 공헌이익률:  34.2%
이번 주 공헌이익률:  31.8%
변화:               -2.4%p

원인:
  상품 믹스 효과:   -1.8%p  (마진 낮은 C신상품 비중 증가)
  단가 변화 효과:   -0.6%p  (할인 프로모션 적용)
```

## AI Interpretation

위 분해 결과를 바탕으로 AI가 원인을 가설화하고 조치를 권장합니다.

Claude에게 다음 구조로 전달합니다:

```
[분석 컨텍스트]
- 기간: {이번주} vs {지난주}
- 채널별 매출 변화: {Step 2 결과}
- 수수료율 변화: {Step 1 결과}
- Top Mover 상품: {Step 3 결과}
- 공헌이익률 변화: {Step 4 결과, 원가 있을 때만}

[요청]
각 변화에 대해:
1. 가능한 원인 (내부/외부 요인 구분)
2. 우선순위 조치 (즉각 / 단기 / 검토)
3. 확인이 필요한 데이터 포인트
```

### 기대 출력 형식

```
🔍 원인 분석 및 조치 권장

━━━ 쿠팡 매출 집중 (수수료율 +1.5%p) ━━━
가능한 원인:
  - 스마트스토어/카페24 광고 예산 축소 → 유입 감소
  - 쿠팡 로켓배송 전환으로 노출 증가
  - 계절적 쿠팡 선호 증가

조치:
  ✅ 즉각: 스마트스토어 쿠폰 발행으로 유입 회복 (목표: 점유율 복원)
  🔄 단기: 카페24 광고 예산 재검토
  📊 검토: 쿠팡 vs 스마트스토어 객단가 비교

━━━ B상품 쿠팡 판매 하락 (-12건) ━━━
가능한 원인:
  - 재고 부족 → 품절 또는 배송 지연으로 노출 감소
  - 경쟁사 신규 진입 또는 가격 인하
  - 리뷰 평점 하락

조치:
  ✅ 즉각: 재고 현황 확인 (`sppt inventory list`)
  🔄 단기: 쿠팡 상품 리뷰 및 경쟁사 가격 모니터링
  📊 검토: 광고비 ROI 재측정
```

## Error Handling

- **원가 미등록**: Step 4 (매출이익률) skip, 나머지 분석은 진행
- **지난 주 데이터 없음**: 2주 이상 데이터로 비교 기간 조정
- **채널 1개만 있음**: 채널 분포 분석 skip, 상품별 분석에 집중
- **`--offset` 미구현**: `--from`/`--to` 날짜로 수동 지정 안내
