# trend-research

트렌드 키워드를 리서치하고 시장 수요를 파악하는 워크플로우.

## When to use

- "이 키워드 검색량 어때?", "뭐가 잘 팔려?", "트렌드 분석해줘"
- 신규 상품 소싱 전 수요 확인
- 키워드 간 비교가 필요할 때
- 전문몰(무신사, 올리브영 등) 랭킹 확인

## Prerequisites

- `sppt` CLI 설치 완료

## Steps

### 1. 키워드 검색량 조회

```bash
sppt trend keyword <키워드>
```

네이버 데이터랩 기반 검색량 + 트렌드 방향을 확인합니다.

판단 기준:
| 월 검색량 | 수요 |
|---|---|
| 10,000+ | 높음 — 경쟁도 높을 수 있음 |
| 1,000~10,000 | 적정 — 니치 시장 가능성 |
| 1,000 미만 | 낮음 — 틈새 or 비인기 |

### 2. 키워드 비교

후보 키워드가 여러 개일 때 상대 비교:

```bash
sppt trend compare <키워드1> <키워드2> [키워드3]
```

검색량, 트렌드 방향, 경쟁강도를 비교합니다.

### 3. 전문몰 랭킹 확인

특정 카테고리의 전문몰 인기 상품을 확인합니다:

```bash
sppt trend platform musinsa
sppt trend platform oliveyoung
sppt trend platform hwahae
```

지원 플랫폼: 무신사, 올리브영, 화해 등

### 4. 결과 종합

사용자에게 다음을 제안합니다:

1. **검색량 충분 + 상승 트렌드**: 소싱 적기 → `/sppt-source-to-register`로 진행
2. **검색량 충분 + 하락 트렌드**: 재고 리스크 있음, 소량 테스트 권장
3. **검색량 부족**: 키워드 변경 또는 연관 키워드로 확장

## Error handling

- **검색량 데이터 없음**: 키워드 오타 확인, 영문/한글 변환 시도
- **전문몰 데이터 실패**: 해당 플랫폼 API 상태 확인

## Notes

- 네이버 데이터랩은 상대값 기반이므로 절대 검색량은 추정치입니다
- 시즌 상품은 전년 동기 대비 확인 권장
- 소싱 전 트렌드 확인은 재고 리스크를 줄이는 핵심 단계입니다
